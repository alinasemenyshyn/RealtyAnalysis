Про проєкт
Цей інструмент автоматично аналізує свіжі оголошення квартир з dom.ria.com на ознаки шахрайства за допомогою великої мовної моделі (Llama 3.3 70B через Groq API). Кожному оголошенню присвоюється відсоток ризику від 0 до 100%, а результати відображаються у вигляді інтерактивних діаграм.
Як це вирішує проблему:
•	До AI: ручна перевірка одного оголошення займає 5–10 хвилин
•	Після AI: 10 оголошень аналізуються за 30 секунд
•	LLM виявляє тонкі ознаки шахрайства: копі-паст тексти, відсутність адреси, тиск на терміновість

⚙️ Встановлення та запуск
1. Клонування репозиторію
git clone https://github.com/alinasemenyshyn/RealtyAnalysis
cd real-estate-scam-detector
2. Створення та активація virtual environment
Windows:
python -m venv venv
venv\Scripts\activate
macOS / Linux:
python3 -m venv venv
source venv/bin/activate
3. Встановлення залежностей
pip install -r requirements.txt
4. Отримання API ключа Groq
1.	Перейдіть на https://console.groq.com
2.	Зареєструйтесь / увійдіть
3.	API Keys → Create API Key → скопіюйте ключ
5. Налаштування .env файлу
Створіть файл .env у кореневій директорії проєкту:
GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
⚠️ ВАЖЛИВО: Ніколи не комітьте .env файл у Git. Переконайтеся що він є в .gitignore.
6. Запуск
Крок 1 — Збір оголошень (створює/оновлює pages.db):
python database.py
Крок 2 — AI-аналіз оголошень (створює/оновлює result_by_AI.db):
python procesing_from_ai.py
Крок 3 — Запуск графічного інтерфейсу:
python app.py
💡 Порада: Кроки 1 і 2 можна запускати за розкладом (наприклад, щодня через cron/Task Scheduler) щоб автоматично отримувати свіжі дані.

🏗 Архітектура та потік даних
Архітектурні рішення:
•	Два окремих SQLite файли — pages.db (сирі дані) та result_by_AI.db (результати AI). Поєднуються через ATTACH DATABASE при побудові діаграм
•	Tool Calling замість звичайного промпту — LLM сам вирішує коли і з якими параметрами викликати функцію отримання даних
•	Threading у GUI — важкі операції не блокують інтерфейс
•	History по датах — кожен запуск procesing_from_ai.py зберігає нові результати з timestamp, старі не перезаписуються

🤖 Логіка AI аналізу
Шкала оцінки шахрайства
Модель оцінює кожне оголошення за такими критеріями:
Ризик	Вердикт	Ознаки
0–20%	Справжнє	Детальний опис, конкретна адреса, розумна ціна, контакти власника
21–50%	Сумнівне	Деякі деталі відсутні, ціна трохи занижена, розмите розташування
51–80%	Підозріле	Немає адреси, немає контактів, аномально низька ціна, копі-паст текст
81–100%	Шахрайство	Тиск на терміновість, вимога передоплати, "надто добре щоб бути правдою"
